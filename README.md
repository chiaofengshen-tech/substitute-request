# 代課詢問 ＆ 追蹤管理

LINE 詢問訊息產生器 + Google Sheets 代課追蹤。  
網站：https://chiaofengshen-tech.github.io/substitute-request/

---

## 快速設定流程

```
建立 Google Sheets → 貼上 Apps Script → 執行 setupSheet()
→ 部署成 Web App → 複製 URL → 填回網站設定
```

---

## 步驟一：建立 Google Sheets

1. 開啟 [Google Sheets](https://sheets.google.com)，新增試算表
2. 命名為：**短代薪水追蹤資料庫**
3. 工作表名稱保持預設（後面 Apps Script 會自動建立「代課追蹤」分頁）

---

## 步驟二：建立 Apps Script

1. 在 Google Sheets 上方選單點「**擴充功能**」→「**Apps Script**」
2. 刪除預設的空白程式碼
3. 將本專案的 `Code.gs` 檔案內容**完整貼上**
4. 按「💾 儲存」（Ctrl+S / Cmd+S）
5. 專案名稱可改為「代課追蹤 API」

---

## 步驟三：執行 setupSheet()

1. 在 Apps Script 編輯器上方，函式下拉選單選「**setupSheet**」
2. 按「▶ 執行」
3. 第一次執行會要求授權：
   - 點「審查權限」→ 選你的 Google 帳戶
   - 看到「Google 尚未驗證此應用程式」→ 點「進階」→「前往（不安全）」→「允許」
4. 執行成功後，Google Sheets 會出現「代課追蹤」工作表，並自動建好欄位和下拉選單

---

## 步驟四：部署成 Web App

1. 在 Apps Script 編輯器右上角點「**部署**」→「**新增部署作業**」
2. 點「類型」旁邊的齒輪圖示 ⚙️ → 選「**網頁應用程式**」
3. 設定如下：
   - 說明：`代課追蹤 API v1`
   - 執行身分：**我（你的 Google 帳戶）**
   - 誰可以存取：**所有人**（Anyone）
4. 點「**部署**」
5. 複製顯示的「**網頁應用程式 URL**」（格式：`https://script.google.com/macros/s/xxxxxxx/exec`）

> ⚠️ 「所有人」表示任何人都能讀寫你的試算表，請妥善保管 URL，不要公開分享。

---

## 步驟五：填回網站設定

1. 開啟 https://chiaofengshen-tech.github.io/substitute-request/
2. 點「⚙️ Google Sheets 連線設定」
3. 貼上剛才複製的 Web App URL
4. 按「儲存」
5. URL 會存在瀏覽器的 localStorage（每台裝置需個別設定一次）

---

## 步驟六：測試連線

1. 點追蹤清單的「↻ 重新整理」
2. 若顯示「連線正常」代表設定成功
3. 填寫一筆代課資料，按「複製詢問訊息並加入追蹤清單」
4. 確認 Google Sheets 有新增一列資料

---

## 更新 Apps Script 後重新部署

修改 `Code.gs` 後，要讓變更生效：

1. 「部署」→「**管理部署作業**」
2. 找到目前版本，點「✏️ 編輯」
3. 版本選「**建立新版本**」
4. 點「部署」（URL 不變）

## 欄位說明

| 欄位 | 說明 |
|------|------|
| id | 系統自動產生 |
| 狀態 | 預設「已詢問」，可在 Sheets 改為「已完成」或「已取消」 |
| 結算月份 | 例：114年6月 |
| 代課類型 | 節代 / 日代 |
| 代課教師 | 代課的老師姓名（自動加「老師」） |
| 日期 | 例：6/23 或 9/1～9/5 |
| 星期 | 例：三 或 一～五（可自動偵測） |
| 節次或日代 | 節代顯示節次，日代顯示「日代」 |
| 班級 | 例：201 |
| 科目 | 節代才有 |
| 授課教師 | 原本上課的老師（自動加「老師」） |
| 代課節數 | 節代時自動預估，可修改 |
| 代課日數 | 日代時預設 1 |
| 導師日數 | 日代時預設 0 |
| 假別 | 月底補填：公、事、病、喪、身心調適假、其他 |
| 摘要 | 月底補填 |
| 備註 | 月底補填 |
| 建立時間 | 系統自動 |
| 更新時間 | 系統自動 |

## 常見問題

**Q：不同裝置（手機 / 家裡電腦 / 公務電腦）都能用嗎？**  
A：可以。資料存在 Google Sheets，只要每台裝置都設定相同的 Web App URL 即可。

**Q：Web App URL 要每台裝置都填嗎？**  
A：是的，URL 存在瀏覽器 localStorage，每台裝置需填一次。

**Q：寫入失敗怎麼辦？**  
A：檢查 Apps Script 是否已部署、「誰可以存取」是否設為「所有人」、URL 是否正確。

**Q：月底假別、摘要、備註可以在 Sheets 直接改嗎？**  
A：可以，直接在 Google Sheets 的「代課追蹤」工作表修改即可。
